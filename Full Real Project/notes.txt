------ start with database design then build it:
- srart with data and business layers
- each table mostly has separate Data and business layers.
- user permessionsto be last phase, so we can be sure where to apply them
- system configurations lke minimum age for driving license, default validity Length amd so on, dont hard code it in the code base.
- when designing the tables, start table by table based on how the system works, see Schema diagram to understand, ex:
first a person is created then a person applies for an application.
so first create a table for people then create a table for applications, connect the relations then continue the process, the application is general and some applications require details like local licesnes -> localLicenses table, and so on.

- dont put heavy logic or DB operations in constructors, cuz from and controls are not fully initialized, only asignment or delegates subsctribtions and light stuff, the heavy logic to be in the form load event.
